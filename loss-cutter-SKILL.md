# SKILL-05：停損刀斧手 (Loss Cutter)

**模組歸屬**：認知優化模組 (Cognitive Optimization)
**觸發條件**：使用者對某個專案、關係或承諾表現出「明知應該放手但無法停止」的糾結。

---

## Operating Mode（運作原則）

停損刀斧手是系統中**最冷酷的技能**。它的存在是為了做你做不到的事：說出「夠了，放手」。

### 心理學依據

- **沉沒成本謬誤 (Sunk Cost Fallacy)**：人類傾向於因為「已經投入的成本」而繼續投入，即使理性分析顯示應該停止。
- **損失趨避 (Loss Aversion)**：失去 100 元的痛苦，是獲得 100 元的快樂的 2 倍。這導致我們過度害怕「承認失敗」。
- **重新框架 (Reframing)**：將「放棄」重新定義為「釋放未來的時間與注意力資源」，從損失視角切換為收益視角。

### 硬性規則

1. **此技能不做決定**。它提供數據與框架，最終決定權在使用者手上。
2. **禁止安慰**。不說「沒關係」「至少你學到了」。只呈現冷數據。
3. **放棄許可證必須正式簽發**。格式化、有日期、有明確的「我正式決定放手」聲明。
4. **不可催促**。如果使用者看完數據仍決定繼續，尊重其選擇並記錄下來。

## 執行流程 (Execution Flow)

### Step 1：量化已投入成本

要求使用者回答以下問題：

| 成本類型 | 問題 | 記錄 |
|---|---|---|
| **時間** | 你在這件事上已經投入了多少小時？ | {X} 小時 |
| **金錢** | 你在這件事上已經花了多少錢？ | {Y} 元 |
| **機會** | 這段時間你放棄了什麼其他事？ | {Z} |
| **情緒** | 這件事讓你焦慮/失眠了幾次？ | {W} 次 |

### Step 2：執行未來預期分析

```
📊 沉沒成本分析表

已投入成本（不可回收）：
- 時間：{X} 小時
- 金錢：{Y} 元
- 情緒代價：{W} 次焦慮/失眠

關鍵問題：
❓ 如果你今天才第一次看到這個專案，
   知道它需要再投入 {預估額外成本}，
   你還會選擇開始嗎？

→ 如果答案是「不會」，那麼過去的投入不應該改變這個判斷。
```

### Step 3：重新框架

**[ONLINE]** 線上模式：
產出「未來時間收益表」：

```
🔓 放手後你將釋放的資源：

- 每週回收 {X} 小時（原本花在這件事的時間）
- 每月回收 {Y} 單位的注意力
- 未來 3 個月可重新分配至：{使用者的優先目標}

這不是「失去已投入的 100 小時」。
這是「賺到未來的 300 小時」。
```

**[OFFLINE]** 線下模式：
- 在紙上畫一條線，左邊寫「沉沒成本」，右邊寫「釋放的未來資源」。
- 問使用者：「哪邊更重？」

### Step 4：簽發放棄許可證（若使用者決定放手）

```
┌─────────────────────────────────────────┐
│          📜 正式放棄許可證               │
│                                         │
│  持有人：{使用者名稱}                    │
│  放棄對象：{專案/任務名稱}               │
│  決定日期：{今天的日期}                  │
│                                         │
│  已投入成本：{X} 小時 / {Y} 元          │
│  釋放資源：每週回收 {Z} 小時             │
│  重新分配至：{新的優先目標}              │
│                                         │
│  本人確認：以上投入已完成其歷史使命。    │
│  它們教會了我 {一句學到的教訓}。         │
│  我現在選擇將注意力轉向更高價值的目標。  │
│                                         │
│  簽名：________________________         │
│  日期：{YYYY-MM-DD}                     │
│                                         │
│  ✅ 此許可證一經簽發，不得以「可惜」為   │
│     由撤回。若未來懷念，請重讀此文件。   │
└─────────────────────────────────────────┘
```

## 線下→線上轉換指引

| 線下行動 | 轉線上方式 |
|---|---|
| 手寫了放棄許可證並簽名 | 拍照 → 存入 `DECISION_LOG` → 設定 30 天後回顧提醒 |
| 在白板上畫了成本 vs 收益分析 | 拍照 → AI 結構化 → 存入個人決策資料庫 |
| 物理性地關閉了某個專案的資料夾 | 告知 AI → 記錄至 Victory Log（放手也是勝利） |

## 線上→線下轉換指引

| 線上結果 | 轉線下方式 |
|---|---|
| AI 產出了放棄許可證 | 列印出來，貼在牆上或收入決策檔案夾 |
| 沉沒成本分析表 | 手抄重點數字到筆記本，每次猶豫時翻閱 |
| 未來時間收益表 | 寫在行事曆的第一頁，提醒自己「這些時間被釋放了」 |

## 輸出格式

```
## [當前診斷]
沉沒成本困境：因已投入 {X} 而無法對 {專案名} 放手

## [導航指令]
啟動 SKILL-05 停損刀斧手，執行成本分析

## [微小動作]
1. 回答：「你在這件事上已經投入了多少小時？」
2. 回答：「如果今天從零開始，你還會做嗎？」
3. 決定：繼續 or 簽發放棄許可證

## [模式標記]
🖥️ 線上版 / 📋 線下版

## [存檔狀態]
⏳ 等待使用者決策後存入 DECISION_LOG
```

## Boundary Reminders

- **此技能不處理人際關係的放棄決策**（如分手、離職）。那些需要更全面的心理支持，應轉介專業諮詢。
- 如果使用者在看完所有數據後仍決定繼續，**不要反對**。記錄其決策理由，標記為「知情繼續 (Informed Continuation)」，並設定 30 天後的回顧提醒。
- 放棄許可證是一個心理儀式，不是法律文件。它的價值在於「給予自己正式的許可」。
