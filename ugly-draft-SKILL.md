# SKILL-03：醜陋初稿 (Ugly Draft)

**模組歸屬**：認知優化模組 (Cognitive Optimization)
**觸發條件**：使用者陷入完美主義迴圈，反覆修改卻無法完成，或遲遲不敢開始「因為怕寫不好」。

---

## Operating Mode（運作原則）

醜陋初稿是一個**反直覺技能**。一般人會以為 AI 應該幫你寫得更好，但這個技能的邏輯是：**幫你寫得更爛**。

### 心理學依據

- **完美主義癱瘓**：當標準設定為「100 分才能交出去」時，0 分到 100 分的落差會觸發杏仁核劫持，導致完全無法動筆。
- **糾錯本能 (Error-Correction Instinct)**：人類看到明顯的錯誤時，會產生強烈的修改衝動。這股衝動比「從零開始創造」的動力強大得多。
- **策略**：先產出 60 分的垃圾 → 糾錯本能自動啟動 → 修改垃圾比從零開始容易 100 倍。

### 硬性規則

1. **禁止在此技能中進行品質評估**。不評分、不評論好壞。
2. **AI 的唯一台詞類型是「這還不夠爛，請再寫爛一點」**。
3. **使用者寫出任何文字都算成功**。即使只寫了「我不知道要寫什麼」，那也是一個合格的醜陋初稿。
4. **限時嚴格 10 分鐘**。10 分鐘一到，無論品質如何，立刻停止並存檔。

---

## 執行流程 (Execution Flow)

### Step 1：宣告垃圾模式

向使用者明確宣告規則：

```
🗑️ 垃圾模式已啟動。

接下來 10 分鐘的規則：
1. 你寫的東西不需要好。事實上，越爛越好。
2. 不要刪除任何字。打錯字不要回頭改。
3. 可以寫「我不知道要寫什麼」——這也算。
4. 10 分鐘後我會強制叫停。

現在請開始寫。主題是：{使用者的目標任務}
```

### Step 2：持續施壓（反向激勵）

**[ONLINE]** 線上模式：
在使用者書寫過程中，每隔 2-3 分鐘插入一次反向激勵：

- 「太好了，但這看起來太工整了。能不能更亂一點？」
- 「你用了完整的句子欸，這不對。試試用注音或斷句。」
- 「我看到你停下來想了。不要想，直接打字。」
- 「完美。我是說，完美地爛。繼續。」

**[OFFLINE]** 線下模式：
- 產出一張「垃圾模式規則卡」讓使用者放在手邊：

```
┌─────────────────────────────┐
│  🗑️ 垃圾模式規則            │
│                             │
│  ✅ 寫錯字：繼續寫          │
│  ✅ 離題了：繼續寫          │
│  ✅ 寫「我不知道」：合格    │
│  ❌ 刪除文字：違規          │
│  ❌ 重新開始：違規          │
│  ❌ 「等我想一下」：違規    │
│                             │
│  ⏱️ 限時 10 分鐘            │
└─────────────────────────────┘
```

### Step 3：10 分鐘強制停止

計時結束後，無論使用者寫了什麼：

1. **立刻讚美**：「你贏了。你現在手上有東西了。從零到有，這是今天最大的勝利。」
2. **存檔**：將醜陋初稿原封不動存入 `DRAFT_LOG`。標記為 `draft_v0_ugly`。
3. **不要修改**：此技能不負責修改。修改是下一個階段的事。

### Step 4：及格線煞車 (Good-Enough Brakes)

如果使用者在後續階段開始修改初稿，啟動「及格線煞車」子機制：

- 事先定義 80 分的及格標準（由使用者自己定義，如「能看懂、有結構、有結論」）。
- 達標後**強制停止修改**：「已達及格線。繼續修改的邊際效益趨近於零。存檔，去做下一件事。」

## 線下→線上轉換指引

| 線下行動 | 轉線上方式 |
|---|---|
| 手寫了一頁亂七八糟的草稿 | 拍照 → OCR → 存為 `draft_v0_ugly.md` |
| 用語音念了一段混亂的想法 | Whisper 語音辨識 → 自動結構化為草稿大綱 |
| 在白板上畫了心智圖 | 拍照 → AI 辨識節點 → 轉為 Markdown 列表 |

## 線上→線下轉換指引

| 線上結果 | 轉線下方式 |
|---|---|
| 在電腦上打了醜陋初稿 | 列印出來，用紅筆圈重點（糾錯本能加倍） |
| AI 產出了垃圾模式規則 | 列印或手抄規則卡，貼在螢幕旁邊 |

## 輸出格式

```
## [當前診斷]
完美主義發作中：任務「{任務名}」卡在「不夠好」的迴圈裡

## [導航指令]
啟動 SKILL-03 醜陋初稿，限時 10 分鐘

## [微小動作]
1. 打開空白文件（或拿出白紙）
2. 開始寫任何跟主題有關的文字（越爛越好）
3. 10 分鐘後回來告訴我「寫完了」

## [模式標記]
🖥️ 線上版 / 📋 線下版

## [存檔狀態]
⏳ 等待使用者完成後，將初稿存為 draft_v0_ugly
```

## Boundary Reminders

- 此技能的產出物（醜陋初稿）**絕不可直接發布或提交**。它只是打破僵局的工具。
- 如果使用者在 10 分鐘內完全無法寫出任何字（連「我不知道」都寫不出來），改為啟動 SKILL-06（思緒傾印），用語音方式繞過書寫障礙。
- 不要在垃圾模式中討論「為什麼你追求完美」。行動優先，分析留到之後。
