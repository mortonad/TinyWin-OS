# SKILL-12：雲端封存 (Cloud Archiver)

**模組歸屬**：外部連結模組 (External Connectivity)
**觸發條件**：當一個階段性的工作結束（如：寫完一份計畫書、產出一段代碼、完成一次深度諮詢），需要正式存檔為辦公格式文件。

---

## Operating Mode（運作原則）

雲端封存是「專案的檔案室」。它強調的是檔案的「正式性」與「結構化」。確保每一份檔案都有明確的元數據（Metadata），方便未來進行全文檢索。

### 心理學依據

- **完結感 (Sense of Completion)**：將流動的對話轉化為固定的 Word/Excel 檔案，能提供強烈的心理完結感，幫助使用者從上一個任務順利切換到下一個。
- **階層化組織 (Hierarchical Organization)**：雲端硬碟的資料夾結構能輔助空間記憶，讓資訊變得更有組織感。

### 硬性規則

1. **必須包含「檔案命名規範」**。格式：`{YYYYMMDD}_{專案名}_{版本}_{內容關鍵字}`。
2. **產出物必須包含時間戳與參與者標記**。
3. **Word 格式用於「文檔/計畫」**；**Excel 格式用於「清單/數據/表格」**。
4. **自動產出「版本紀錄」**。若為重複封存，需詢問是否覆蓋或建立新版本（v2, v3...）。

---

## 執行流程 (Execution Flow)

### Step 1：選定檔案格式

當偵測到封存需求時：

```
📁 雲端封存模式已啟動。

我將為你準備正式檔案，請選擇目標格式：
1. 📄 **Microsoft Word** (適合長文計畫、會議紀錄、文章)
2. 📊 **Microsoft Excel** (適合待辦清單、預算表、數據比較)
3. 📑 **PDF** (適合正式合約或不可修改的報告)
```

### Step 2：結構化內容處理

**[ONLINE]** 線上模式：

AI 根據選定格式，產出包含以下標記的內容：

- **[TIME_STAMP]**: {YYYY-MM-DD HH:MM}
- **[USER_PROMPT]**: {使用者的關鍵指令}
- **[AI_RESPONSE]**: {AI 的核心產出}
- **[METADATA]**: {標籤、專案代號}

### Step 3：上傳至 Google Drive

**[ONLINE]** 線上模式：
- 自動生成檔案內容。
- 若具備相關 API 權限，則上傳至 `My Drive/FlowGPS_Archives/{YYYY}/{MM}/`。
- 否則提供檔案下載連結或內容預覽，引導使用者手動存入。

**[OFFLINE]** 線下模式：
- 提供「封存檢查清單」：指示使用者確認檔案名稱、目錄分類與權限設定。

## 輸出格式

```
## [當前診斷]
專案階段性完結：偵測到高品質產出物，啟動正式封存程序以建立數位資產庫。

## [導航指令]
啟動 SKILL-12 雲端封存，限時 3 分鐘。

## [微小動作]
1. 選擇檔案格式 (Word/Excel/PDF)
2. 確認檔案命名與存放路徑
3. 點擊「上傳至雲端」或「下載存檔」

## [模式標記]
🖥️ 線上版 / 📋 線下版

## [存檔狀態]
⏳ 等待檔案生成中
```
