# SKILL-09：對話存檔 (Chat Logger)

**模組歸屬**：外部連結模組 (External Connectivity)
**觸發條件**：使用者需要將對話內容持久化保存，以便未來參考、編輯或在不同平台上匯入。

---

## Operating Mode（運作原則）

對話存檔是「知識的備份碟」。對話紀錄不應只存在於雲端對話視窗中，應將其轉化為可隨時查閱、搜尋、且格式一致的本地檔案。

### 心理學依據

- **外部化認知 (Externalized Cognition)**：將資訊從流動的對話轉移到穩定的文件，能減輕大腦的維持壓力，讓使用者更專注於思考而非記憶。
- **回顧效應 (Review Effect)**：擁有一份乾淨的對話筆記，能提高未來重新查閱與應用的機率。

### 硬性規則

1. **存檔內容必須包含「日期、主題、參與者」**。
2. **預設使用 Markdown (.md) 格式**，因為它最兼具易讀性與跨平台相容性。
3. **區分「對話全文」與「重點摘要」**。除非使用者要求全文，否則預設匯出結構化摘要 + 關鍵問答。
4. **禁止存檔無意義的雜談**。若對話內容過短且無實質內容，提醒使用者：「內容過於簡略，建議累積更多洞察後再存檔。」

---

## 執行流程 (Execution Flow)

### Step 1：確認存檔範圍

當觸發信號出現時，確認使用者要存檔的內容範圍：

```
💾 對話存檔模式已啟動。

請選擇你要備份的範圍：
1. 📝 **結構化摘要**（最推薦：包含洞察與行動清單）
2. 📜 **完整對話紀錄**（包含所有問答細節）
3. 🎯 **僅特定部分**（請說明範圍）
```

### Step 2：格式化處理

**[ONLINE]** 線上模式：

AI 將內容整理為以下格式：

```markdown
# 📝 對話存檔：{主題}
- 日期：{YYYY-MM-DD}
- 來源：TinyWin OS (FlowGPS)

---

## 💡 核心洞察
- {洞察 1}
- {洞察 2}

## 🎯 決策與行動
- [ ] {待辦 1}
- [ ] {待辦 2}

---

## 💬 關鍵對話紀錄
**使用者**：{關鍵問題}
**AI**：{關鍵回答}

---
*存檔於：{時間}*
```

### Step 3：匯出與存檔

**[ONLINE]** 線上模式：
- 自動生成檔案並提供內容預覽。
- 詢問存放路徑（預設：`e:/morton-company-x-flow-ceo/對話摘要/`）。

**[OFFLINE]** 線下模式：
- 提供「存檔指令卡」：指示使用者如何複製當前對話並貼入自己的記事本應用程式（如 Notion, Obsidian 或手機備忘錄）。

---

## 輸出格式

```
## [當前診斷]
對話價值積累中：偵測到高品質對話，啟動存檔機制防止數位資產流失。

## [導航指令]
啟動 SKILL-09 對話存檔，限時 2 分鐘。

## [微小動作]
1. 選擇「摘要」或「全文」
2. 確認檔案內容預覽
3. 執行「寫入本地」或「複製到剪貼簿」

## [模式標記]
🖥️ 線上版 / 📋 線下版

## [存檔狀態]
⏳ 等待存檔指令執行
```
