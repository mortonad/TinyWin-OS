# SKILL-07：進度防腐 (Breadcrumb Pause)

**模組歸屬**：RAM 外包模組 (Memory Outsourcing)
**觸發條件**：使用者正在執行某項任務時被外部事件打斷，需要暫時離開當前工作。

---

## Operating Mode（運作原則）

進度防腐借鏡遊戲中的「存檔點」概念：你不需要打完整個關卡才能存檔。隨時存檔，隨時讀檔。

### 心理學依據

- **注意力殘留 (Attention Residue)**：當你從任務 A 切換到任務 B 時，一部分注意力仍然「殘留」在任務 A 上。這會降低你在任務 B 上的表現。
- **蔡格尼克效應 (Zeigarnik Effect)**：未完成的任務會持續佔用工作記憶。中斷的任務比完成的任務更容易被記住——但這種記憶是焦慮性的，不是建設性的。
- **閉環策略 (Closure Strategy)**：即使任務未完成，也可以透過「記錄下一步」來給大腦一個「偽完成」信號，從而釋放注意力殘留。

### 硬性規則

1. **封裝必須在 2 分鐘內完成**。中斷時沒時間寫長篇日記。
2. **只記錄 3 件事**：做到哪了、下一步是什麼、回來時第一個動作。
3. **不要在封裝過程中「順便」處理其他事**。封裝就是封裝，不是工作。
4. **封裝完成後，必須告訴大腦「這已經被保存了」**，讓蔡格尼克效應解除。

---

## 執行流程 (Execution Flow)

### Step 1：偵測中斷信號

當使用者說出以下類型的話語時，立刻啟動封裝：

- 「等一下，有人找我」
- 「我要先去處理另一件事」
- 「先暫停，我等等回來」
- 「今天做到這裡」

### Step 2：快速封裝（2 分鐘）

立刻詢問使用者三個問題（或從上下文自動推斷）：

```
⏸️ 進度封裝啟動（限時 2 分鐘）

請回答三個問題，或直接說「幫我記」讓我從對話中自動抓取：

1. 📍 你剛才做到哪裡了？
   （例：報告寫到第二段）

2. ➡️ 下一步本來要做什麼？
   （例：要查一個數據來源）

3. 🚀 回來時的第一個動作是什麼？
   （例：打開 Google Scholar 搜尋 XX）
```

**[ONLINE]** 線上模式：
- 若使用者說「幫我記」，AI 自動從當前對話/文件的上下文中提取以上三項。
- 封裝結果存入 `BREADCRUMB_LOG`，標記時間戳。

**[OFFLINE]** 線下模式：
- 產出一張「暫停卡」：

```
┌─────────────────────────────────┐
│  ⏸️ 暫停存檔點                  │
│  時間：{HH:MM}                  │
│                                 │
│  📍 做到：                      │
│  _______________________________│
│                                 │
│  ➡️ 下一步：                    │
│  _______________________________│
│                                 │
│  🚀 回來時第一動作：            │
│  _______________________________│
│                                 │
│  ✅ 大腦，這已經被保存了。      │
│     你可以放心去處理其他事了。  │
└─────────────────────────────────┘
```

### Step 3：閉環確認

封裝完成後，向使用者明確宣告：

```
✅ 進度已封裝。

你的工作狀態已經被安全保存。
你的大腦現在可以 100% 切換到下一件事。
回來的時候告訴我「讀檔」，我會把你的進度還給你。
```

### Step 4：讀檔重啟

當使用者回來並說「讀檔」或「我回來了」時：

```
📂 讀取存檔點 [{時間戳}]

📍 上次做到：{進度描述}
➡️ 下一步：{待執行事項}
🚀 現在請執行：{第一個動作}

歡迎回來。你的脈絡已恢復。開始吧。
```

---

## 線下→線上轉換指引

| 線下行動 | 轉線上方式 |
|---|---|
| 在紙上寫了暫停卡的三個欄位 | 拍照 → OCR → 存入 `BREADCRUMB_LOG` |
| 語音備忘錄「我做到___，下一步是___」 | Whisper → 結構化 → 自動建立讀檔點 |
| 在筆記本角落潦草記了幾個關鍵字 | 回到電腦後口述補充 → AI 整合為完整存檔 |

## 線上→線下轉換指引

| 線上結果 | 轉線下方式 |
|---|---|
| AI 產出了封裝摘要 | 列印或手抄到便利貼，貼在螢幕/鍵盤上 |
| BREADCRUMB_LOG 有讀檔點 | 隔天早上先看便利貼上的「第一個動作」再開電腦 |
| 多個暫停點累積 | 週末統一檢視，將超過 3 天未讀檔的項目轉入正式待辦 |

## 輸出格式

### 封裝時：
```
## [當前診斷]
工作中斷：任務「{任務名}」被外部事件打斷

## [導航指令]
啟動 SKILL-07 進度防腐，限時 2 分鐘封裝

## [微小動作]
1. 回答：做到哪了？下一步？回來第一動作？
2. 確認封裝內容
3. 安心切換至其他事務

## [模式標記]
🖥️ 線上版 / 📋 線下版

## [存檔狀態]
✅ 已存入 BREADCRUMB_LOG [{時間戳}]
```

### 讀檔時：
```
## [讀檔恢復]
📍 上次做到：{進度}
➡️ 下一步：{待執行}
🚀 現在第一動作：{具體動作}

脈絡已恢復。開始吧。
```

## Boundary Reminders

- 如果使用者的中斷是「我今天不想做了」而非被外部打斷，不要強制封裝。改為引導使用者做一個「今日小結」，然後啟動 SKILL-01（睡前掃雷）。
- 封裝的內容應該簡短到「10 秒就能讀完」。如果封裝結果超過 5 行，需要精簡。
- 如果使用者累積超過 5 個未讀檔的暫停點，建議進行一次 SKILL-06（思緒傾印），統一清理所有懸掛任務。
