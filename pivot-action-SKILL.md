# SKILL-02：壓力降級 (Pivot Action)

**模組歸屬**：啟動防禦模組 (Initiation Defense)
**觸發條件**：使用者面對任務時出現「想動但動不了」的癱瘓狀態，或對任務規模感到壓迫。

---

## Operating Mode（運作原則）

壓力降級是本系統**最泛用的技能**，也是路由器的預設降級目標。核心機制：大腦欺騙術。

### 心理學依據

- **杏仁核劫持 (Amygdala Hijack)**：當任務被感知為「龐大且不確定」時，杏仁核會發出威脅信號，導致前額葉皮質（負責規劃與執行）關閉。
- **繞過策略**：將任務的「感知威脅」降到杏仁核的觸發閾值以下。具體手段是將任務縮小到 1/10，讓大腦判定「這不需要害怕」。
- **多巴胺駭客**：一旦完成縮小版任務，大腦會釋放多巴胺。這股多巴胺會自動驅動使用者繼續執行，進入「滾雪球」狀態。

### 硬性規則

1. **降級後的任務必須 ≤ 10 分鐘**。如果超過，繼續降級。
2. **降級後的任務必須有「可見產出」**（一個檔案、一行字、一張圖），不可是純思考性任務。
3. **禁止在降級過程中討論「為什麼你會焦慮」**。不分析原因，直接行動。
4. **使用者完成降級版任務後，禁止立刻加碼**。先存檔、先慶祝，再問「要繼續嗎？」。

---

## 執行流程 (Execution Flow)

### Step 1：捕獲原始任務

詢問或從上下文中提取使用者當前焦慮的任務。

- 使用者說：「我要寫完整個專案報告，但一直拖延。」
- 捕獲到的原始任務：`寫完整個專案報告`

### Step 2：執行 1/10 降級

使用以下降級公式：

```
降級版 = 原始任務的第一個物理動作 + 時間上限 5~10 分鐘
```

**降級範例矩陣**：

| 原始任務 | ❌ 仍然太大 | ✅ 降級版 |
|---|---|---|
| 寫完專案報告 | 寫報告大綱 | 打開 Word，寫出 3 個小標題 |
| 找工作 | 更新履歷 | 打開 LinkedIn，改一行自我介紹 |
| 整理房間 | 清理書桌 | 把書桌上的 3 樣東西放回原位 |
| 學一門新技術 | 看完教學影片 | 打開教學頁面，看前 5 分鐘 |
| 回覆那封困難的信 | 想好怎麼回 | 打開信件，先打「Hi，感謝你的來信」 |

**[ONLINE]** 線上模式：
- 直接在聊天中產出降級版任務。
- 若可存取檔案系統，自動建立對應的空白文件（如 `報告_草稿.docx`）。

**[OFFLINE]** 線下模式：
- 產出一張「行動卡片」格式：

```
┌─────────────────────────────┐
│  🎯 5 分鐘挑戰              │
│                             │
│  動作：打開 Word，寫 3 個標題 │
│  限時：⏱️ 5 分鐘            │
│  完成判定：螢幕上有 3 行字   │
│                             │
│  □ 完成了！                 │
└─────────────────────────────┘
```

### Step 3：啟動計時器

**[ONLINE]**：建議使用者設定手機計時器 5 分鐘。
**[OFFLINE]**：指示使用者翻轉手機螢幕朝下，用實體計時器或鐘錶計時。

### Step 4：完成後的多巴胺封裝

任務完成後，**不要立刻加碼**。執行以下封裝流程：

1. **存檔**：將完成的微小產出存入 `EXECUTION_LOG` 或手寫在筆記本上。
2. **慶祝**：明確告訴使用者「這就是一個勝利。你剛剛從零到一了。」
3. **詢問**（不強迫）：「你現在感覺如何？想繼續下一個 10 分鐘嗎？還是先停在這裡？」

## 線下→線上轉換指引

| 線下行動 | 轉線上方式 |
|---|---|
| 手寫了 3 個標題在紙上 | 拍照 → OCR 辨識 → 存入數位草稿 |
| 口頭說「我完成了整理書桌」 | Whisper 語音辨識 → 寫入 Victory Log |
| 便利貼上打了勾 | 下次開啟 AI 時口述回報 → 更新進度追蹤 |

## 線上→線下轉換指引

| 線上分析結果 | 轉線下方式 |
|---|---|
| AI 產出了降級版任務 | 列印或手抄到便利貼上，貼在工作區域 |
| AI 設定了計時器提醒 | 改用實體鬧鐘，手機完全收起來 |
| AI 建立了空白草稿文件 | 先在紙上手寫，寫完再打字進電腦 |


## Boundary Reminders

- 如果使用者的任務本身就很小（如「回一封簡訊」），不需要降級。直接說「這個任務本身就是降級版，請直接執行」。
- 如果使用者在 5 分鐘後仍然卡住，不要責備。切換至 SKILL-03（醜陋初稿）嘗試用「寫爛一點」的方式打破僵局。
- 此技能處理的是「啟動問題」，不是「持續問題」。如果使用者能啟動但無法持續，改用 SKILL-07（進度防腐）。
