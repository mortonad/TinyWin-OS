# SKILL-10：社群橋接 (Community Bridge)

**模組歸屬**：外部連結模組 (External Connectivity)
**觸發條件**：使用者希望將在 AI 對話中達成的共識、待辦事項或瞬間靈感，立即同步到日常使用的社群軟體中。

---

## Operating Mode（運作原則）

社群橋接是「行動的廣播站」。將封閉的 AI 對話轉向開放的協作或個人紀錄空間，增加行動的「可見性」與「儀式感」。

### 心理學依據

- **社會承諾 (Social Commitment)**：即使是傳到私人的 Discord 頻道，這種「發布」的動作也會在心理上強化對任務的承諾感。
- **環境觸發 (Environmental Trigger)**：在常用的通訊軟體（Line/Discord）看到紀錄，能更有效地觸發下一步行動。

### 硬性規則

1. **訊息必須精簡**。社群平台不適合閱讀長篇大論，訊息長度應控制在一個手機螢幕內。
2. **必須標註「時間」與「狀態」**。例如：`[ tinywin-os ] [ 2026-05-14 ] 任務更新：...`。
3. **區分「私人紀錄」與「團隊分享」**。詢問使用者發布的目標頻道或對象。
4. **禁止自動連發**。每一則發布都必須經過使用者的「發布確認」，防止錯誤資訊流出。

---

## 執行流程 (Execution Flow)

### Step 1：選定同步內容

當觸發信號出現時，抓取當前對話中最具行動價值的片段：

```
🌐 社群橋接模式已啟動。

我已準備好以下訊息摘要：
---
🚀 **今日小贏：{主題}**
✅ 完成：{完成事項}
📅 接下來：{下一步}
💡 關鍵：{一句话洞察}
---
你想將它同步到哪裡？
1. 🔵 **Discord** (指定 Webhook 或頻道)
2. 🟢 **Line** (傳送到個人聊天室或群組)
3. 🟡 **其他終端** (複製格式化文字自行貼上)
```

### Step 2：傳輸處理

**[ONLINE]** 線上模式：
- AI 格式化為適合 Discord Embed 或 Line 訊息的排版。
- 若有配置 Webhook，則自動執行傳送；否則提供複製按鈕。

**[OFFLINE]** 線下模式：
- 指示使用者：「將剛才的摘要截圖或複製，貼入你的社群群組『置頂筆記』中。」

### Step 3：同步回報

傳送完成後，在本地紀錄中標記同步狀態：

```
✅ 已成功推播至 {平台}。
🔗 關聯紀錄已標註同步標籤。
```

---

## 輸出格式

```
## [當前診斷]
行動可見化：將數位洞察轉移至日常通訊環境，強化認知觸發。

## [導航指令]
啟動 SKILL-10 社群橋接，限時 1 分鐘。

## [微小動作]
1. 確認同步平台 (Discord/Line/Other)
2. 檢視訊息預覽
3. 點擊「確認發布」或「複製訊息」

## [模式標記]
🖥️ 線上版 / 📋 線下版

## [存檔狀態]
⏳ 等待同步確認
```
