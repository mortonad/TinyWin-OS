# SKILL-08：想法萃取 (Idea Harvester)

**模組歸屬**：知識萃取模組 (Knowledge Harvest)
**觸發條件**：使用者在 AI 對話、閱讀文章、觀看影片後，產生了有價值的想法或洞察，需要在遺忘前結構化保存。

---

## Operating Mode（運作原則）

想法萃取是「記憶的急救站」。大多數人跟 AI 聊完之後，當下覺得「超有收穫」，隔天卻只記得「好像聊了什麼，但想不起來了」。這個技能在想法還熱騰騰的時候就把它真空包裝。

### 心理學依據

- **艾賓浩斯遺忘曲線 (Ebbinghaus Forgetting Curve)**：新學到的資訊在 20 分鐘後遺忘 42%，1 小時後遺忘 56%，1 天後遺忘 74%。不在當下記錄，基本上就等於沒發生。
- **生成效應 (Generation Effect)**：自己用語言重新表述的資訊，記憶保留率比單純閱讀高 3-5 倍。因此萃取過程要求使用者用「一句話」重新描述洞察，而非只是複製貼上。
- **間隔重複 (Spaced Repetition)**：結構化存檔後，可搭配定期回顧機制，將短期記憶轉化為長期知識。

### 硬性規則

1. **萃取必須在 5 分鐘內完成**。超過 5 分鐘就變成了「整理」，不是「萃取」。
2. **每次萃取不超過 5 個重點**。如果超過，代表對話太密集，需要分次萃取。
3. **每個洞察必須附帶「所以呢？」**——即這個想法對我的下一步行動意味著什麼。沒有行動意涵的純知識，標記為「📚 參考」而非「💡 洞察」。
4. **禁止萃取模糊感受**（如「今天聊得很有收穫」）。必須是具體的、可複述的知識點。

## 執行流程 (Execution Flow)

### Step 1：啟動萃取模式

當偵測到對話即將結束，或使用者表達「該記一下」的意圖時：

```
🌾 想法萃取模式已啟動。

我來幫你從剛才的對話中挖礦。
請回答以下問題，或說「自動萃取」讓我從對話中直接提取：

1. 💡 最重要的 1-3 個洞察是什麼？
2. 🎯 因此你接下來要做什麼？
3. ❓ 還有什麼問題是未解決的？
```

### Step 2：萃取與分類

**[ONLINE]** 線上模式：

若使用者說「自動萃取」，AI 自動掃描當前對話，提取並分類：

```
🌾 萃取報告 | {日期} {時間}
📌 來源：{對話主題 / 平台 / 文章標題}

━━━ 💡 關鍵洞察 (Key Insights) ━━━
1. {洞察描述}
   → 所以呢：{行動意涵}
   → 標籤：#{分類標籤}

2. {洞察描述}
   → 所以呢：{行動意涵}
   → 標籤：#{分類標籤}

━━━ 🎯 行動項目 (Action Items) ━━━
1. [ ] {具體待辦}（截止：{建議時間}）
2. [ ] {具體待辦}（截止：{建議時間}）

━━━ ❓ 未解問題 (Open Questions) ━━━
1. {問題描述}
   → 下次要問誰 / 查什麼：{方向}

━━━ 📚 參考資料 (References) ━━━
1. {值得收藏的連結、工具名稱、書籍}

━━━ 🔗 關聯 (Connections) ━━━
→ 與「{過去某個專案/想法}」有關聯

━━━ 📊 萃取統計 ━━━
```

### Step 3：存檔至指定位置

**[ONLINE]** 線上模式：
- 自動將萃取報告存入指定的本地路徑或雲端位置。
- 預設存檔路徑結構：

```
knowledge-vault/
├── {YYYY}/
│   ├── {MM}/
│   │   ├── {YYYY-MM-DD}__{主題關鍵字}.md
│   │   └── {YYYY-MM-DD}__{主題關鍵字}.md
│   └── monthly-review.md
└── _index.md          # 自動生成的索引
```

- 存檔格式為 Markdown，包含完整的 YAML frontmatter：

```yaml
---
date: "2026-05-08"
time: "22:45"
source: "ChatGPT 對話"
topic: "TinyWin OS 技能開發"
tags: [AI, prompt-engineering, CBT]
status: "raw"         # raw → reviewed → integrated
---
```

**[OFFLINE]** 線下模式：
- 指示使用者拍照/語音回傳萃取卡。
- AI 轉為數位格式後存入相同的路徑結構。

### Step 4：觸發回顧提醒

萃取完成後，根據內容重要性設定回顧週期：

```
✅ 萃取完成。已存入：knowledge-vault/2026/05/2026-05-08__tiny-win-skills.md

📅 回顧排程：
- 明天早上 → 快速掃一眼今天的洞察（1 分鐘）
- 本週日 → 週回顧時整合到專案計畫中
- 月底 → 月度知識盤點時檢視是否仍相關

要我設定提醒嗎？
```

## 萃取模板庫 (Template Library)

根據不同的對話類型，使用不同的萃取模板：

### 模板 A：技術學習萃取

```
🔧 技術筆記 | {主題}
- 問題：{要解決什麼}
- 解法：{核心做法}
- 程式碼片段：{關鍵 code}
- 踩坑：{注意事項}
- 延伸：{可以進一步學的}
```

### 模板 B：商業/策略洞察

```
💼 策略筆記 | {主題}
- 機會：{發現了什麼機會}
- 驗證方式：{怎麼確認這是真的}
- 下一步：{最小可行動作}
- 風險：{可能的阻礙}
- 連結：{跟哪個現有專案有關}
```

### 模板 C：創意/靈感萃取

```
✨ 靈感筆記 | {主題}
- 原始靈感：{想到了什麼}
- 觸發原因：{為什麼會想到}
- 初步形態：{如果做成產品/內容會長什麼樣}
- 缺什麼：{還需要什麼才能實現}
- 保鮮期限：{這個靈感的時效性}
```

### 模板 D：對話要點萃取（通用）

```
📝 對話摘要 | {來源}
- 主題：{聊了什麼}
- 共識：{達成了什麼結論}
- 歧異：{還沒對齊的觀點}
- 待辦：{誰要做什麼}
- 下次議程：{下次要接著討論什麼}
```

## 線下→線上轉換指引

| 線下行動 | 轉線上方式 |
|---|---|
| 在萃取卡上手寫了洞察 | 拍照 → OCR → AI 存入 knowledge-vault |
| 聽完 Podcast 口述了幾個重點 | Whisper 語音轉文字 → AI 套用萃取模板 → 存檔 |
| 讀書時畫了重點拍照 | 圖片 OCR → AI 提取標註內容 → 結構化存檔 |
| 開會時在筆記本上速記 | 回到電腦後口述重點 → AI 補完並歸檔 |

## 線上→線下轉換指引

| 線上結果 | 轉線下方式 |
|---|---|
| AI 萃取了 3 個關鍵洞察 | 寫在便利貼上，貼在工作區最醒目的位置 |
| 行動項目被提取出來 | 寫入紙本待辦清單，或貼在螢幕邊框上 |
| 未解問題被標記 | 寫在隨身小本子上，遇到適合的人就問 |

## Boundary Reminders

- 此技能處理的是「知識萃取」，不是「筆記整理」。如果使用者已有大量累積筆記需要重新整理，改用 SKILL-06（思緒傾印）處理存量，SKILL-08 只處理增量。
- 如果對話內容很短或很簡單（如只是問了一個事實性問題），不需要啟動萃取。直接告訴使用者「這次對話不需要萃取，內容很簡單。」
- 萃取的品質比數量重要。寧可只記 1 個真正有用的洞察，也不要記 10 個模糊的感想。
- 若使用者連續多天都在萃取但從不回顧，應提醒：「你已經累積 {N} 份萃取報告未回顧。要不要花 10 分鐘做一次快速整合？」
- 如果使用者的萃取內容涉及別人的商業機密或隱私，提醒使用者注意資訊的適當處理。
